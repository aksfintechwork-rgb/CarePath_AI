STEP 1: FUNCTIONAL QC (CORE FEATURES)

For EACH FEATURE, verify:

Does it work exactly as intended?

Does it fail silently anywhere?

What happens if input is missing / wrong / partial?

What happens if user clicks fast / repeats actions?

Deliver output as:

Feature Name:
Expected Behavior:
Actual Behavior:
Issues Found:
Severity: (Critical / Major / Minor)

STEP 2: USER FLOW QC (REAL HUMAN BEHAVIOR)

Test flows like a real user, not a developer.

For each role:

First-time user

Returning user

Power user

Confused user

Check:

Can user complete task without guidance?

Any unclear buttons or labels?

Any misleading confirmations?

Any irreversible actions without warning?

STEP 3: EDGE CASE & FAILURE TESTING (VERY IMPORTANT)

Actively TRY TO BREAK THE APP:

Empty inputs

Extremely large inputs

Rapid clicks / duplicate submissions

Network failure mid-action

Page refresh during critical step

Partial save states

Document exact failure points.

STEP 4: DATA INTEGRITY & CONSISTENCY

Verify:

Data saved = data displayed = data retrieved

No mismatch between UI, API, and database

No duplicate records created

No orphan records

Ask:

What happens if save succeeds but UI fails?

What happens if API fails but UI shows success?

STEP 5: AI / AUTOMATION QC (IF APPLICABLE)

If AI is used, verify:

Output format consistency

No hallucination

No invented data

No missing required fields

Safe fallback when AI fails

Human approval gates exist

Flag any place where AI output could:

Mislead user

Auto-execute without approval

Override human decision
